

---

CPU 旨在处理复杂的串行任务。它面向分支逻辑多、数据依赖性强的代码。其目标是缩短单个任务的执行时间。

而 GPU 旨在处理大规模的并行任务。它面向结构统一、计算密集型的任务，如图形渲染、深度学习和大规模物理模拟。其目标是在单位时间内完成尽可能多的计算。

CPU 的特征是二进制向下兼容与 ISA 标准的公开。CPU 的软件接口由 ISA 定义。只要支持相同的 ISA，不同厂商的 CPU 均可运行同一程序。此外，同一厂商的 ISA 演进极为稳健，通常仅做增量扩展而极少删减。由于 ISA 手册完全公开，第三方开发者可以独立于 CPU 厂商开发编译器。这种高度的稳定性与开放性，使得 CPU 程序能以机器码二进制的形式直接分发并跨代运行。

相比之下，GPU 则以驱动和图形/计算 API 为接口。首先，不同厂商的 GPU ISA 互不兼容且大多互不公开；其次，同一厂商的架构演进也常有剧变（如 AMD 从 VLIW 转向 SIMD）。为了弥合硬件差异，厂商通过驱动程序适配各类图形与计算 API（如 DirectX、Vulkan、OpenCL 等）。因此，GPU 程序通常以源代码（如 HLSL、GLSL）或机器无关的中间表示（如 SPIR-V、DXIL）形式分发，最终由驱动程序在运行前或运行时动态编译为目标硬件的特定指令。

值得注意的是，这与以字节码等形式分发CPU程序，并由运行时环境动态编译（如 CLR）或提前编译（如 ART）以实现跨 ISA 平台不谋而合。

然后，来看内部的不同。CPU 中，相当大的晶片面积被用于控制单元。真正的算术逻辑单元（ALU）占比相对较小。这是为了通过复杂的机制发现串行指令间的指令级并行性，以进行加速。相反，GPU 面向的任务使其程序天生就是数据级并行的，GPU 可以将大量的面积用于计算，而小部分面积用于控制。例如，RDNA4 中，每 32 或 64 个 work-item 共用一个 PC（而通过 EXECute Mask 实现条件执行）。

此外，CPU内部有相当大面积用于缓存。AMD RDNA4 架构中，最大的 RX 9070（356.5 mm^2）也只不过配备了 8MB L2 cache 和 64 MB L3 cache，相比之下，Intel Core Ultra 100 Series 的 Compute tile（70 mm^2）即配备有 14MB L2 和 24MB L3。

GPU 的流水线相对 CPU 更深，因为 CPU 往往有复杂的分支逻辑，需要短流水线得到较小的分支预测惩罚；而 GPU 则需要长流水线来隐藏延迟。

从存储器层次结构来看，CPU 方面有主存、各级 Cache（其中，对高性能 CPU 来说 L1、L2 往往是 CPU 核心私有的，而 L3 和可能的 L4 是共享的，部分 CPU 甚至只有两级 Cache），以及最终的寄存器。而 GPU，以 RDNA4 为例，一方面可以访问主存，另一方面有自己的显存，有共享的 L2、L1 缓存，而每个 Workgroup Processor 又有自己的指令缓存、常量缓存、纹理缓存，以及所谓 Local Data Share。最后，Workgroup Processor 内部的 Compute Unit 的 Vector ALU Unit 又有各自的寄存器。

最终，GPU 可以获得远高于 CPU 的浮点计算性能（特别地，某些嵌入式 CPU 甚至根本不支持硬件浮点）。
