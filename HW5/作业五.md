# 异构计算 作业五

> 采用图像存储器，编写二维图像旋转的 OpenCL 程序。基本要求：
>
> 1. 参考资料 \[1\] 中提供了样例代码，编写 OpenCL Kernel；
> 2. 编写相同功能的 CPU 程序；
> 3. 选取至少两个不同尺寸的图像，对比 CPU 程序和 OpenCL 程序执行时间。

构建一个动态链接库，暴露两个函数：

- `rotate_image_cpu`：使用 CPU 旋转图片。
- `rotate_image_opencl`：使用 OpenCL（GPU）旋转图片。

构建一个 F# 程序，使用 ImageSharp 库承担读取/写入图片的任务。亦即：

1. 解析命令行参数，确定输入、输出图片的路径，确定旋转角度，以及使用的引擎（OpenCL/CPU）；
2. 读取图片；转换为 32 位 RGBA 格式；
3. 启动上述动态链接库的函数；
4. 动态链接库返回，F# 端保存图片。

选取 Libbie 和 Microsoft Wallpaper 为输入图像。格式为 PNG。图片顺时针旋转 60 度；运行 10 次取平均值。

> 前者为 LibreOffice 吉祥物，后者为 Windows Insider 5 周年纪念壁纸。

首先，我们测量程序完整的运行时间：

|图片|尺寸|时间（OpenCL）|时间（CPU）|
|---|---|---|---|
|Libbie|$2391 \times 2816$|581.60 ms|511.06 ms|
|Microsoft Wallpaper|$4092 \times 2098$|1318.720 ms|1248.35 ms|

可见，因为涉及内存复制、F# 调用 DLL、以及 OpenCL 编译的时间，OpenCL 的整体用时甚至比 CPU 长。

---

然后是仅计算部分的时间（CPU 取 Native 函数执行时间；OpenCL 仅取 Kernel 的执行时间）：

|图片|尺寸|时间（OpenCL）|时间（CPU）|
|---|---|---|---|
|Libbie|$2391 \times 2816$|3.84 ms|70.02 ms|
|Microsoft Wallpaper|$4092 \times 2098$|4.50 ms|94.55 ms|

此时可见 OpenCL 的性能优势。
