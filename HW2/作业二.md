## 环境配置

安装 Intel oneAPI。因为我没有安装完整的 Visual Studio，只安装了构建工具，此后的工作需要在命令行中进行。

我使用 Xmake 为构建系统。配置文件 `xmake.lua` 如下：

```lua
set_optimize("fastest")
set_languages("cxx20")
set_warnings("all", "extra")
set_encodings("utf-8")
add_rules("plugin.compile_commands.autoupdate")

local function add_intel_opencl()
    add_includedirs("C:\\Program Files (x86)\\Intel\\oneAPI\\compiler\\latest\\include")
    add_linkdirs("C:\\Program Files (x86)\\Intel\\oneAPI\\compiler\\latest\\lib")
    add_links("OpenCL")
end

target("homework")
    set_kind("binary")
    add_files("src/HW-1.cxx") -- 添加源文件
    add_intel_opencl()
```

`add_includedirs` 和 `add_linkdirs` 配置了编译器寻找头文件、库文件的路径，`add_links("OpenCL")` 要求链接 OpenCL。

编译时，首先配置 Xmake 使用 Intel icx 编译：

```bash
xmake f -p windows --cxx=cl@icx
```

如果手动编译（以 Intel icx 为例）：

```bash
icx <源文件> /std:c++20 /O3 /I 'C:\Program Files (x86)\Intel\oneAPI\compiler\latest\include' /link /libpath:'C:\Program Files (x86)\Intel\oneAPI\compiler\latest\lib' OpenCL /o <输出>
```

> 6. 查询设备核心工作频率并输出；
> 7. 查询设备全局内存大小并输出；
> 8. 对输出的平台信息和设备信息进行说明。

详细实现见代码。输出如下：

```text
Platform 0: Intel(R) OpenCL Graphics
    Device 0: Intel(R) Arc(TM) Graphics
        Type: GPU
        Version: OpenCL 3.0 NEO
        Compute Units: 112
        Clock: 2200 MHz
        Global Memory: 16814 MB
```

这表明：机器上有两个 OpenCL 平台，各包含一个设备。即 CPU（Intel Core Ultra）和 GPU（Intel Arc）。

对于 CPU，它支持 OpenCL 3.0，有 18 个计算单元（即 CPU 最多的线程数）。总计 32 GB 内存。最大时钟频率为 0，原因未知。

对于 GPU，它支持 OpenCL 3.0 NEO。查阅[文档](https://github.com/intel/compute-runtime)可知“NEO”指“Intel Graphics Compute Runtime”。GPU 有 112 个计算单元（对应 112 个 Xe Vector Engines），约 17 GB 内存。最大时钟频率 2200 MHz。
